# ai-codyssey

## B1-1 미션

### 업무 자동화 프롬프트 패키지 — 고객 문의 이메일 자동 분류 및 답변 초안 생성

중소 규모 이커머스·서비스 기업 고객지원팀(CS)을 대상으로, 고객 문의 이메일을
**자동으로 분류하고 답변 초안을 생성**하는 LLM 기반 업무 자동화 프롬프트 패키지를
설계·검증한 결과물이다. 모델 선정 → 시스템 설계 → 실행 검증의 3단계(Phase)로 구성된다.

#### 📁 산출물 구성

| 파일 | 단계 | 내용 |
|------|------|------|
| `B1-1/Phase1_LLM모델비교선정보고서.docx` | Phase 1 | LLM 모델 비교·선정 |
| `B1-1/Phase2_시스템설계문서.docx` | Phase 2 | 시스템·프롬프트 설계 |
| `B1-1/Phase3_실행로그.docx` | Phase 3 | 실행 로그 및 검증 |

---

### Phase 1 — LLM 모델 비교·선정

동일한 업무 과업 프롬프트를 **GPT-4o, Claude Sonnet, Gemini 1.5 Pro** 3종 모델에
입력하여 7개 항목(5점 만점)으로 평가하였다.

- **평가 기준**: 정확도, 문체·톤 자연스러움, 지시 준수율, 처리 속도, 비용 효율,
  컨텍스트 유지, 한국어 품질
- **테스트 케이스**: 환불 요청(TC-01), 배송 지연 항의(TC-02), 제품 사용법 복합 문의(TC-03)

| 모델 | 종합 점수 |
|------|:---------:|
| **Claude Sonnet** ✅ | **4.57** |
| GPT-4o | 4.43 |
| Gemini 1.5 Pro | 4.29 |

**최종 선정: Claude Sonnet (4.57 / 5.0)**
- 지시 준수율 만점 — 분류 규칙·답변 형식을 가장 충실히 이행
- 15턴 이상 장기 대화에서도 컨텍스트 드리프트 없음
- 한국 비즈니스 경어체 답변 품질 우수
- GPT-4o 대비 약 40% 저렴한 비용 효율

---

### Phase 2 — 시스템 설계 문서

선정된 Claude Sonnet 기반으로 타겟 사용자·페르소나·분류 체계·프롬프트를 설계하였다.

- **페르소나**: CS 팀 리더 "김지영" (일 100건 이상 이메일 처리, 응대 품질 균일화 목표)
- **이메일 분류 체계 (6종)**: CAT-01 환불·반품 / CAT-02 배송 / CAT-03 제품 불량 /
  CAT-04 사용법·기술 지원 / CAT-05 계정·결제 / CAT-06 기타 (각 우선순위 부여)
- **프롬프트 개선 (v1 → v2)**
  - 자유 텍스트 → **JSON 구조 강제** (`category_code`, `priority`, `sentiment`, `confidence`, `draft_reply`)
  - 카테고리명만 나열 → CAT 코드 + 우선순위 명시
  - 어조 규칙(경어체·공감 표현·번호 목록) 명시
  - **Few-shot 예시 3종** 제공 → 분류 정확도 88% → 96% 향상
- **검증 전략**: 분류 정확도(≥95%), 형식 일관성(파싱 오류 0), 답변 품질(≥4.0/5),
  응답 속도(≤5초), 엣지 케이스(오류율 ≤5%)

---

### Phase 3 — 실행 로그 및 검증

v2 확정 프롬프트로 **13턴**의 실제 업무 시나리오를 수행하며 분류·답변을 검증하고,
발견된 문제를 수정하였다. (목표 10턴 초과 달성)

#### 발견·수정한 주요 문제

| 턴 | 유형 | 문제 | 수정 |
|----|------|------|------|
| T4 | 🔶 주의 | 영어 이메일에 영어로 답변 | "답변은 반드시 한국어" 규칙 추가 |
| T7 | ⚠️ 환각 | 타 턴 주문번호를 임의 연결 | 미제공 정보는 고객에게 확인 요청 |
| T12 | ⚠️ 집계 오류 | 동일 이메일(T4/5) 중복 집계 | 고유 이메일 건수 기준 재집계 |

#### 검증 결과 요약

| 지표 | 목표 | 실제 |
|------|:----:|:----:|
| 전체 처리 턴 수 | ≥ 10턴 | 13턴 |
| 분류 정확도 (1차) | ≥ 95% | 92.3% |
| 분류 정확도 (수정 후) | ≥ 95% | 100% |
| JSON 파싱 성공률 | 100% | 100% |
| 평균 응답 품질 | ≥ 4.0 | 4.2 |

#### 향후 개선 권고

1. "답변은 반드시 한국어로 작성" 규칙 명시 (T4 재발 방지)
2. "이메일 내 명시된 정보만 사용, 없는 정보는 고객에게 확인 요청" 규칙 추가 (T7 환각 방지)
3. "대화 이력 요약 시 고유 이메일 건수 기준 집계" 지침 추가 (T12 오류 방지)
4. 실운영 전 엣지 케이스 20건 이상 추가 테스트 (욕설·혼합 언어·초장문 이메일)
