# LLM 모델 비교·선정 보고서

**업무 자동화 프롬프트 패키지 — Phase 1**

- **과업:** 고객 문의 이메일 자동 분류 및 답변 초안 생성
- **작성일:** 2025년 6월

---

## 1. 평가 개요

동일한 업무 과업 프롬프트를 GPT-4o, Claude Sonnet, Gemini 1.5 Pro 3종 모델에 입력하여 성능을 비교하고 최적 모델을 선정한다.

### 1.1 평가 목적

- 고객 문의 이메일 분류·답변 자동화에 최적화된 LLM 모델 선정
- 모델별 정확도·비용·한국어 품질을 객관적 기준으로 비교
- 선정 근거를 문서화하여 이후 시스템 설계의 기반 마련

### 1.2 평가 기준 (7개 항목, 5점 만점)

| # | 항목 | 설명 |
|---|------|------|
| 1 | 정확도 | 이메일 카테고리 분류 정확성 |
| 2 | 문체·톤 자연스러움 | 한국 비즈니스 이메일 맥락 적합성 |
| 3 | 지시 준수율 | 프롬프트 지시 사항 이행 정도 |
| 4 | 처리 속도 | 응답 생성까지 소요 시간 |
| 5 | 비용 효율 | 토큰당 비용 대비 성능 |
| 6 | 컨텍스트 유지 | 장기 대화에서 맥락 보존 능력 |
| 7 | 한국어 품질 | 한국어 표현의 자연스러움과 정확성 |

## 4. 최종 선정 결론

### ✅ 최종 선정 모델: Claude Sonnet (4.57 / 5.0)

### 선정 근거

- **지시 준수율 5점 만점** — 시스템 프롬프트의 분류 규칙·답변 형식을 가장 충실히 이행
- **컨텍스트 유지력 최상** — 15턴 이상 장기 대화에서도 맥락 드리프트 없음

- **비용 효율 균형** — GPT-4o 대비 40% 저렴하면서도 정확도는 동등 수준

### 모델별 탈락 사유

- **GPT-4o** — 비용 부담이 크며 컨텍스트 유지력이 Claude 대비 낮음

## 6. 심층 인터뷰 — 비용 제약 시나리오 대응 전략

> **질문:** 만약 비용 제약으로 '무료 모델만' 써야 한다면 품질을 유지하기 위해 프롬프트/프로세스를 어떻게 바꿀 것인가?

### 6.1 전제

유료 모델(Claude Sonnet) 대신 무료 또는 무료 티어 모델(예: Gemini 1.5 Flash 무료 티어, Llama 3 계열 오픈소스 모델 등)을 사용해야 하는 상황을 가정한다. 무료 모델은 일반적으로 **지시 준수율 저하, 형식 드리프트, 컨텍스트 유지력 약화**가 발생하므로, 모델 성능 부족분을 **프롬프트 설계와 프로세스 설계로 보완**하는 것이 핵심 전략이다.

### 6.2 프롬프트 차원의 보완 전략

| 전략 | 내용 | 보완하는 약점 |
|------|------|--------------|
| **Few-shot 확대 (3개 → 6개)** | 카테고리 6종 전부에 대해 입력→출력 예시를 1개씩 제공하여 분류·형식을 예시로 고정 | 지시 준수율 저하, 형식 드리프트 |
| **출력 형식 단순화** | JSON 필드를 핵심 4개(category_code, priority, sentiment, draft_reply)로 축소하고 필드별 허용값을 열거형으로 제한 | 복잡한 스키마 이행 실패 |
| **지시문 분해 (1프롬프트 → 2단계)** | "분류"와 "답변 초안 생성"을 별도 프롬프트로 분리 호출. 단순 과업일수록 무료 모델 정확도가 유지됨 | 복합 지시 누락 |
| **명시적 금지 규칙 강화** | "입력에 없는 주문번호·정책 수치를 만들어내지 말 것"을 부정 지시 + 위반 예시로 명시 | 환각 빈도 증가 |
| **자기 검증 단계 삽입** | 출력 직전 "JSON 유효성과 카테고리 코드가 CAT-01~06 범위인지 스스로 점검 후 출력" 지시 추가 | 형식 오류 |

### 6.3 프로세스 차원의 보완 전략

1. **검증 레이어 추가:** 모델 출력 직후 코드 레벨에서 JSON 파싱 + 필드 값 검증을 수행하고, 실패 시 자동 재호출(최대 2회). 모델 품질 저하를 시스템이 흡수한다.
2. **신뢰도 기반 라우팅:** confidence < 0.7 건은 자동 발송 후보에서 제외하고 담당자 수동 검토 큐로 배정한다. 무료 모델에서는 사람 검토 비율을 의도적으로 높여(약 20~30%) 품질을 방어한다.
3. **키워드 사전 분류 병행:** "환불", "불량" 등 고신뢰 키워드 룰을 1차 필터로 두고, 룰과 모델 분류가 불일치하면 사람 검토로 라우팅한다 (하이브리드 분류).
4. **답변 템플릿 라이브러리화:** 무료 모델에는 자유 생성 대신 카테고리별 사전 승인 템플릿의 빈칸 채우기(slot-filling)만 맡겨, 품질 하한선을 보장한다.

### 6.4 결론

무료 모델 전환 시 **"모델이 덜 똑똑한 만큼 과업을 더 잘게 쪼개고, 검증을 시스템과 사람에게 더 많이 맡긴다"**는 원칙으로 전환한다. 이 경우 처리 속도와 자동화율(완전 자동 발송 비율)은 낮아지지만, 고객에게 전달되는 최종 답변 품질은 유지할 수 있다.
