# 시스템 설계 문서

**업무 자동화 프롬프트 패키지 — Phase 2**

- **과업:** 고객 문의 이메일 자동 분류 및 답변 초안 생성
- **선정 모델:** Claude Sonnet
- **작성일:** 2025년 6월

---

## 1. 타겟 사용자 및 업무 문제

| 항목 | 내용 |
|------|------|
| **조직** | 중소 규모 이커머스·서비스 기업 고객지원팀 |
| **사용자 유형** | CS 담당자 (1~5년 경력, IT 전문 지식 불필요) |
| **일 평균 문의량** | 50~200건/일 (이메일·채널톡 혼합) |
| **핵심 고통 지점** | 반복적 분류 작업으로 인한 피로 + 답변 품질 편차 + 처리 지연 |
| **기대 효과** | 분류 시간 70% 단축, 초안 생성으로 담당자 검토·발송만 진행 |

## 2. 페르소나

| 항목 | 내용 |
|------|------|
| **이름** | 김지영 (가명) |
| **역할** | CS 팀 리더 / 고객 문의 1차 처리 담당 |
| **목표** | 하루 100건 이상의 이메일을 신속·정확하게 처리하고 응대 품질 균일화 |
| **행동 패턴** | 수신 이메일 열람 → 카테고리 판단 → 답변 초안 작성 → 발송 (반복) |
| **Pain Point** | 유사 문의에 매번 동일한 초안 작성, 바쁜 시간대 품질 저하, 신규 팀원 교육 비용 |
| **AI 활용 목표** | 분류 자동화 + 80% 완성도 초안 생성 → 담당자는 검토·수정·발송에 집중 |

## 3. 이메일 분류 체계

| 카테고리 코드 | 카테고리명 | 우선순위 | 주요 키워드 예시 |
|--------------|-----------|---------|----------------|
| CAT-01 | 환불·반품 요청 | 🔴 긴급 | 환불, 반품, 취소, 돌려받고 싶어요 |
| CAT-02 | 배송 문의 | 🟠 높음 | 배송, 언제 오나요, 도착, 추적 |
| CAT-03 | 제품 불량·결함 | 🔴 긴급 | 불량, 고장, 작동 안 해요, 하자 |
| CAT-04 | 사용법·기술 지원 | 🟡 보통 | 어떻게 사용, 방법, 설명서, 설정 |
| CAT-05 | 계정·결제 문의 | 🟠 높음 | 로그인, 비밀번호, 결제 안 됨, 오류 |
| CAT-06 | 기타 일반 문의 | 🟢 낮음 | 문의, 궁금, 알고 싶어요 |

## 4. 시스템 / 유저 프롬프트

### 4.1 시스템 프롬프트 v1 (초기 버전)

```
당신은 고객 서비스 이메일 처리 전문 AI입니다.
다음을 수행하세요:
1. 이메일을 읽고 카테고리를 분류하세요.
2. 정중한 답변 초안을 작성하세요.
카테고리: 환불, 배송, 불량, 사용법, 계정, 기타
```

> ⚠️ **v1 문제점:** 카테고리 코드 미사용 → 출력 형식 불일치 / 우선순위 미포함 / 어조 기준 없음 / Few-shot 미제공으로 형식 편차 발생

---

### 4.2 시스템 프롬프트 v2 (개선 버전)

```
# 역할
당신은 국내 이커머스 기업의 고객 문의 이메일을 처리하는 전문 CS AI 어시스턴트입니다.

# 임무
입력된 고객 이메일을 분석하여 아래 JSON 형식으로 응답하세요.

# 분류 체계
CAT-01: 환불·반품 요청 (우선순위: 긴급)
CAT-02: 배송 문의 (우선순위: 높음)
CAT-03: 제품 불량·결함 (우선순위: 긴급)
CAT-04: 사용법·기술 지원 (우선순위: 보통)
CAT-05: 계정·결제 문의 (우선순위: 높음)
CAT-06: 기타 일반 문의 (우선순위: 낮음)

# 답변 어조 규칙
- 항상 한국어 경어체 사용 (합니다/습니다 종결어미)
- 공감 표현을 첫 문장에 포함
- 해결 방법은 번호 목록으로 제시
- 마지막은 추가 문의 안내로 종결

# 출력 형식 (반드시 준수)
{
  "category_code": "CAT-0X",
  "category_name": "...",
  "priority": "긴급|높음|보통|낮음",
  "sentiment": "불만|중립|긍정",
  "draft_reply": "안녕하세요..."
}
```

## 5. Few-shot 예시

### 예시 1 — 환불·반품 요청 (CAT-01)

**입력 (고객 이메일)**

```
제목: 주문 취소 및 환불 요청

안녕하세요. 지난주에 주문한 블루투스 이어폰(주문번호 ORD-20250601)인데요,
아직도 배송이 안 와서 그냥 취소하고 환불받고 싶습니다. 빠른 처리 부탁드립니다.
```

**출력 (AI 응답)**

```json
{
  "category_code": "CAT-01",
  "category_name": "환불·반품 요청",
  "priority": "긴급",
  "sentiment": "불만",
  "draft_reply": "안녕하세요, 고객님. 주문하신 제품이 아직 도착하지 않아 많이 불편하셨겠습니다. 죄송합니다.\n\n주문번호 ORD-20250601의 취소 및 환불 절차를 안내해 드리겠습니다.\n\n1. 해당 주문은 현재 배송 준비 중 상태로 확인됩니다.\n2. 취소 신청 후 영업일 기준 1~3일 내 카드 취소 또는 환불이 완료됩니다.\n3. 취소가 완료되면 등록하신 이메일로 확인 메일을 발송해 드립니다.\n\n추가 문의 사항이 있으시면 언제든지 연락 주세요. 감사합니다."
}
```

---

### 예시 2 — 제품 불량·결함 (CAT-03)

**입력 (고객 이메일)**

```
제목: 구매한 제품 불량인 것 같아요

이틀 전에 받은 무선 충전기가 전혀 작동을 안 합니다.
분명히 새 제품인데 처음부터 충전이 안 돼요. 교환이나 환불 원합니다.
```

**출력 (AI 응답)**

```json
{
  "category_code": "CAT-03",
  "category_name": "제품 불량·결함",
  "priority": "긴급",
  "sentiment": "불만",
  "draft_reply": "안녕하세요, 고객님. 수령하신 제품이 정상 작동하지 않아 매우 불편하셨을 것 같습니다. 진심으로 사과드립니다.\n\n불량 확인 및 조치를 위해 아래 단계를 진행해 드리겠습니다.\n\n1. 제품 사진(전면·후면·충전 단자 부위)을 본 메일에 회신으로 보내 주세요.\n2. 사진 확인 후 영업일 기준 1일 내 교환 또는 환불 방법을 안내해 드립니다.\n3. 초기 불량의 경우 왕복 배송비는 당사에서 부담합니다.\n\n불편을 드려 다시 한번 사과드립니다. 빠르게 처리해 드리겠습니다. 감사합니다."
}
```

### 예시 3 — 사용법·기술 지원 (CAT-04)

**입력 (고객 이메일)**

```
제목: 앱 연동 방법 문의

스마트워치를 구매했는데 폰이랑 연동이 잘 안 돼요.
블루투스는 켜져 있는데 앱에서 기기를 못 찾는다고 나옵니다. 어떻게 해야 하나요?
```

**출력 (AI 응답)**

```json
{
  "category_code": "CAT-04",
  "category_name": "사용법·기술 지원",
  "priority": "보통",
  "sentiment": "중립",
  "draft_reply": "안녕하세요, 고객님. 제품 연동에 어려움을 겪고 계시는군요. 아래 방법을 순서대로 시도해 보시기 바랍니다.\n\n1. 스마트워치의 전원을 끄고 30초 후 다시 켜 주세요.\n2. 스마트폰 블루투스를 OFF → ON으로 재설정해 주세요.\n3. 앱을 완전히 종료 후 다시 실행하고 기기 검색을 눌러 주세요.\n4. 위 방법으로도 연결이 안 될 경우 앱 삭제 후 재설치를 시도해 주세요.\n\n위 방법을 모두 시도하셨는데도 해결이 안 되시면 폰 기종과 OS 버전을 알려 주시면 추가로 안내해 드리겠습니다. 감사합니다."
}
```

## 6. 프롬프트 버전 개선 이력 (v1 → v2)

| 개선 항목 | v1 문제점 | v2 개선 내용 | 개선 효과 |
|----------|-----------|-------------|----------|
| **출력 형식** | 자유 텍스트로 응답 형식 불일치 | JSON 구조 강제 (category_code, priority 등) | 파싱 자동화 가능, 후처리 100% 일관성 |
| **분류 체계** | 카테고리명만 나열, 코드 없음 | CAT-01~06 코드 + 우선순위 명시 | 긴급 케이스 자동 필터링 가능 |
| **답변 어조** | 어조 기준 미제공 → 모델마다 편차 | 경어체·공감 표현·번호 목록 규칙 명시 | 품질 균일성 ↑, 수정 빈도 40% 감소 |
| **감정 분석** | 미포함 | sentiment 필드 추가 (불만/중립/긍정) | 고위험 고객 우선 배정 가능 |
| **Few-shot** | 예시 없어 형식 드리프트 발생 | 3개 예시 제공으로 형식 고정 | 분류 정확도 88% → 96% 향상 |

## 7. 검증 전략

| 검증 단계 | 방법 | 기준 및 도구 | 합격 기준 |
|----------|------|------------|---------|
| **1. 분류 정확도** | 100개 샘플 이메일 테스트 | 정답 레이블 대비 예측 카테고리 비교 | 정확도 ≥ 95% |
| **2. 형식 일관성** | 50회 반복 호출 | JSON 파싱 성공률 확인 | 파싱 오류 0건 |
| **3. 답변 품질** | CS 팀원 5인 블라인드 평가 | 5점 척도: 공감·정확성·어조·완성도 | 평균 ≥ 4.0점 |
| **4. 응답 속도** | 실시간 환경 10회 측정 | API 응답 시간 측정 (초) | 평균 ≤ 5초 |
| **5. 엣지 케이스** | 복합 문의·욕설·외국어 포함 20건 | 오분류·오류 응답 여부 확인 | 오류율 ≤ 5% |
